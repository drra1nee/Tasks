# Платформа обработки задач (Лабораторная работа №1 семестр 2)

Запуск демонстрации работы платформы осуществляется из корневой директории командой:

```bash
python -m src.main
```

Тестирование всех компонентов реализовано с использованием `pytest`:

```bash
pytest
```

Для проверки процента покрытия тестами используйте:

```bash
pytest --cov=src --cov-report=term-missing
```
---

### Структура проекта

- `src/main.py`: точка входа, демонстрирует работу платформы и проверку контрактов
- `src/contracts/`:
  - `protocol.py`: описание контракта `TaskSource` с использованием `typing.Protocol`
- `src/models/`:
  - `task.py`: класс `Task` (dataclass), представляющий задачу
- `src/sources/`:
  - `generator_source.py`: реализация источника задач `GeneratorTaskSource`
- `tests/`:
  - `test_main.py`: тесты точки входа
  - `test_generator_source.py`: тесты генератора задач и соблюдения контракта
  - `test_modules.py`: тесты класса `Task`
  - `test_protocol.py`: тесты контракта

### Поддерживаемые сущности и контракты

#### Задача (`Task`)
- Представляет задачу платформы
- Поля:
  - `id`: уникальный идентификатор
  - `payload`: произвольные данные задачи
- Поддерживает:
  - `to_dict()`: сериализация в словарь
  - `from_dict()`: десериализация из словаря
  - Сравнение и хеширование на основе `id`

#### Контракт источника (`TaskSource`)
- Описан через `typing.Protocol` с декоратором `@runtime_checkable`
- Требует реализации метода:
  - `get_tasks() -> Iterator[Task]`: возвращает список задач
- Проверка соблюдения контракта:
  - `validate_source(source)`: runtime-проверка через `isinstance`

#### Источники задач
- **Генератор задач (`GeneratorTaskSource`)**:
  - Реализует контракт `TaskSource` без явного наследования
  - Инициализируется количеством задач (`count`) и шаблоном данных (`payload_template`)
  - Поддерживает типы шаблонов:
    - `str`: строка с добавлением индекса (`"task_0"`)
    - `dict`: словарь с добавлением ключа `index`
    - `list`: список с добавлением индекса в конец
    - `int`: преобразуется в строку с добавлением индекса
  - Функция `create_generator_source`: упрощает создание с предустановленными шаблонами

### Особенности реализации

- **Контрактное программирование**:
  - Источники задач не наследуют общий базовый класс
  - Используется `typing.Protocol` для статической проверки типов
- **Runtime-проверки**:
  - Протокол помечен как `@runtime_checkable` для возможности проверки через `isinstance` во время выполнения
  - Функция `validate_source` явно проверяет соответствие источника контракту
- **Генерация данных**:
  - Уникальность id задач обеспечивается глобальным счетчиком `_task_counter`
  - Payload генерируется динамически на основе шаблона и индекса итерации
- **Типизация**:
  - Все публичные интерфейсы имеют аннотации типов
